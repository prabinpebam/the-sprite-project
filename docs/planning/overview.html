The delivery path

Phase 1 · Prove the foundation Verified

Build a headless technical slice using a deliberately small, legally verified LPC asset subset.

  1. Lock the contracts

    Define the smallest versioned project, pack, animation, token, provenance, and export schemas needed by the slice.

  2. Prove composition

    Load real pack data, resolve compatibility and z-order, and compose idle and walk frames deterministically.

  3. Prove palette mapping

    Map source PNG colors to semantic ramp positions and verify project and character token overrides.

  4. Prove delivery

    Export a spritesheet, animation metadata, and exact attribution from the same recipe.

Exit gate: one recipe can be rendered and exported headlessly from real pack data with no license or palette information supplied by UI code.

Execution source of truth

Use Agentic MVP Execution for work-package dependencies, unattended defaults, evidence requirements, recovery rules, and the four bounded owner inputs.

Phase 2 · Complete the vertical slice Verified

Wrap the proven core in the complete MVP user journey: local projects, character selection, immediate preview, token controls, deterministic save/reopen, generic export, and a Godot adapter.

Exit gate: a new user can create a character and use it in Godot without manual frame slicing or hand-written attribution.

Phase 3 · Validate the MVP Verified

Add a structurally independent second pack, test real project recovery and migration, improve workflow friction, and observe developers integrating exported characters into actual game projects.

Exit gate: the second pack requires no renderer changes, saved projects reproduce reliably, and target users complete the character-to-engine journey without assistance.

MVP evolution · Dual-host local-first Specified

UPD-001 evolves the verified baseline into two complementary hosts over one product core: a GitHub Pages web workspace using IndexedDB and local preferences, and a portable Electron Windows workspace using project folders and direct export directories. It includes project archives, migration from current localStorage persistence, exact cross-host parity, secure Electron IPC, and fresh regression evidence for affected MVP flows.

Exit gate: all 11 new web, desktop, cross-host, and shared flows and all 40 objective behaviors pass; affected baseline flows regress cleanly; project and export parity are exact; no severity 0-2 issue remains.

Phase 4 · Open the content ecosystem Next

Only after the pack contract survives the MVP, add pack validation and authoring workflows, broaden humanoid content, and introduce the next engine export adapter. Creatures and mounts follow when their distinct animation and compositing requirements can be modeled from real content.

Exit gate: content can grow independently of application releases, with provenance intact.

Phase 5 · Expand from characters to worlds

Build the environment and terrain generator as the second real producer. Use it to discover which project, token, pack, animation, and export concepts are truly shared. Add structures and props next, followed by the composition surface once multiple producers need to be verified together.

Exit gate: characters and world assets coexist in one project, share a theme, and export through one pipeline without producer-specific duplication in the shared core.

Phase 6 · Complete the studio

Add the advanced systems whose value depends on a healthy multi-producer project: manual pixel and animation editing, material and lighting maps, live engine links, CLI/API automation, and eventually cloud collaboration.

Exit gate: the complete North Star journey works from world generation through detailed editing and automated engine delivery.

Sequencing rules

Working product over planning volume
Create only the specification needed to unblock the active phase.
Exit gates over dates
Advance when the product claim is demonstrated, not because a calendar milestone arrived.
Second implementation before abstraction
Use another real pack or producer to test whether a shared contract is actually shared.
Provenance from day one
Never postpone licensing and attribution to a cleanup phase.
North Star protects direction
Deferred does not mean abandoned; it means sequenced behind evidence.
