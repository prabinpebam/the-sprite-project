Product update · UPD-004

Multi-character Projects

Maintain a bounded cast of independently named and serialized character recipes inside one local-first project.

TL;DR

3 promises, 6 scenarios, 12 capabilities, 7 task flows, and 25 objective behaviors define this update. Status: implemented.

Canonical local run · RUN-UPD004-001

Up to 16 character recipes create, switch, edit, duplicate, rename, delete, save, recover, transfer, and export through public UI. Separate IndexedDB records and folder files remain one atomic project revision; sibling recipes and terrain stay isolated.

Delivered promises

CP-CHARACTER-COLLECTION

Keep a named cast in one project and choose the active editing/export target.

CP-CHARACTER-ISOLATION

Edit and serialize each recipe without changing siblings or terrain.

CP-CHARACTER-PORTABILITY

Reopen and transfer the complete cast across web and Electron.
