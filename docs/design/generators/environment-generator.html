Environment & Terrain Generator

The character generator's counterpart for the ground itself - grass, dirt, water, floors, and paths - built as smart, autotiling terrain rather than loose images.

TL;DR

Where the character generator composes layers, the environment generator composes tiles. The core idea: users pick a material (grass, dirt, water, stone floor, sand, path) and the tool produces a complete, seamless autotile set - edges, corners, and transitions between materials - all recolored from the same palette tokens. Output is a ready-to-use tileset plus the metadata an engine's tilemap needs.

Grounding

Terrain sits on the same 32px grid and perspective as everything else - see the LPC Revised Style Guide for the material rules (wood, stone, plants, water) this generator should encode.

The unit of terrain: materials and transitions

A game floor is never one tile - it's a material that must tile seamlessly and blend into its neighbors. The generator's job is to produce, from a single material choice, every tile needed for that material to sit convincingly in a map.

Base materials

Grass, dirt, sand, stone/rock, water, wood floor, tile floor, path - each a token-driven material with texture rules.

Autotile set

Center, edges, outer/inner corners generated so the material tiles seamlessly in any shape (e.g. blob/47-tile or wang sets).

Transitions

Blends between materials - grass→dirt, sand→water shorelines, path→grass edges - generated as transition tiles.

How generation works

  1. Choose materials

    Pick one or more materials and their palette tokens (a "biome" is a set of materials that go together).

  2. Generate the tile matrix

    The tool synthesizes the full autotile matrix per material - center, edges, corners - using texture rules (grain, noise, foliage) from the style guide.

  3. Generate transitions

    For each pair of adjacent materials, produce transition/blend tiles so borders read naturally.

  4. Scatter & detail

    Optional decorators - pebbles, flowers, cracks, ripples - placed with controllable density for variation.

  5. Export tileset + rules

    Emit the packed tileset image plus autotile/terrain metadata the engine's tilemap can consume directly.

Animated terrain

Water and FX are animated tiles

Some materials aren't static - water ripples, waterfalls, and shoreline foam are multi-frame tiles. The environment generator treats animation as a first-class property of a material, reusing the same animation export path as characters so an engine gets an animated tileset, not just a still image.

User-facing capabilities

  • User can generate a seamless autotile set for any material from one choice.
  • User can generate transition tiles between any two materials.
  • User can assemble a "biome" - a coherent group of materials that blend together.
  • User can control detail/scatter density and variation.
  • User can recolor materials via shared palette tokens (grass to autumn, water to swamp).
  • User can preview terrain on a test map (paint materials, see autotiling live).
  • User can export a packed tileset plus autotile/terrain rules for the target engine.
  • User can output matching normal/specular maps per tile so terrain lights dynamically (see material & lighting maps).
  • User gets merged credits for any source art the materials draw on.

Why this belongs in the same product

Terrain is where most of a 2D world's surface area comes from, and it must match the characters standing on it. Because it shares the grid, perspective, palette tokens, animation path, and export pipeline, it is the natural second generator - not a separate tool.
