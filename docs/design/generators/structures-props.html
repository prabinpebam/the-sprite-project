Structures & Props Generator

Houses, buildings, furniture, and objects - assembled from modular parts in the LPC three-quarter perspective, filling the space between terrain and characters.

TL;DR

The third generator builds objects with volume: a house from walls + roof + door + windows, furniture and clutter as placeable props. It works like the character generator (modular parts, palette tokens, correct z-order) but adds perspective-correct assembly - respecting the style guide's pixel ratios - and outputs multi-tile objects with the footprint/anchor data a scene needs.

Grounding

Structures must obey the style guide's perspective and pixel ratios (1:2 diagonals, 4:3 Y/Z squares) and material rules for wood, stone, metal, and glass.

Two content classes

Structures

  • Buildings: houses, shops, towers, walls, fences
  • Assembled from parts: base, walls, roof, door, windows, trim
  • Multi-tile footprints with an anchor and collision hint
  • Interior vs. exterior variants

Props

  • Furniture: tables, chairs, beds, chests, shelves
  • Clutter: barrels, crates, pots, signs, lamps
  • Nature props: trees, bushes, rocks, stumps
  • Single- or few-tile, often with a shadow + anchor

Modular structure assembly

A building is not one sprite - it is a kit of parts combined under perspective rules. The generator exposes the parts and assembles them so the result is always geometrically consistent.

Shell

Footprint and wall height define the box; the tool draws walls and the roof in the correct 4:3 / 1:2 ratios.

Openings & trim

Doors, windows, chimneys, awnings, signs - snapped to valid positions on the shell.

Materials

Wall/roof/trim materials (wood, stone, thatch, tile) driven by palette tokens and the style guide's texture rules.

Props as a pack, like characters

Props reuse the character generator's pack model: a prop pack is a set of objects with palette bindings, an anchor/footprint, an optional animation (a flickering lamp, a swaying tree), and license metadata. Trees are the bridge to terrain - a "nature" pack shared between environment and props.

What a structure/prop carries

Footprint
The tile area it occupies, for placement and collision hints.
Anchor / origin
The base point that sits on the ground, so props stand correctly in a scene.
Z-order / depth
How it sorts against characters (walk behind a roof, in front of a fence base).
Palette tokens
Recolorable materials, shared with every other generator.
Optional animation
Lamp flicker, water wheel, banner sway - via the shared animation path.
License metadata
Per-part credits, merged on export.

User-facing capabilities

  • User can assemble a building from parts (shell, roof, door, windows, trim) that stay perspective-correct.
  • User can place and recolor props (furniture, clutter, nature) from packs.
  • User can set a footprint, anchor, and z-order so objects drop into a scene correctly.
  • User can give a prop an optional animation.
  • User can recolor structures/props via the shared palette tokens.
  • User can export multi-tile objects with placement metadata for the target engine.
  • User can output normal/specular/emissive maps per object (e.g. glowing windows, glossy metal) - see material & lighting maps.
  • User gets merged credits across all parts used.

Why this completes the set

Characters + terrain + structures/props is the full vocabulary of a 2D scene. Sharing the pack model, palette tokens, perspective rules, and export path means the three generators are variations on one engine - not three products bolted together.
