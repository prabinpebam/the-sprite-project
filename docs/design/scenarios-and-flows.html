Design · Experience

Scenarios, Capabilities & Task Flows

The proposed product told from the user's side - who uses it, everything they can do, and the step-by-step flows that stitch the generators, tokens, editor, maps, and export together.

TL;DR

This page synthesizes the whole design into experience terms: 8 end-to-end scenarios across personas, a consolidated "user can…" catalog spanning every proposal, and detailed task flows for the key journeys - from "assemble a hero" to "build a coherent, dynamically-lit scene and live-sync it into Godot." It is the connective tissue that shows the parts working as one product.

Scope

Covers the proposals documented so far: the character, environment, and structures/props generators; the composition model, token coherence, manual editor, and material & lighting maps.

Personas

Jamie - the jammer

Solo dev on a 48-hour clock. Needs a playable look now; polish is optional. Values speed and defaults over control.

Sam - the solo indie

Programmer building a full game without an artist. Needs a whole world - characters, terrain, buildings - coherent and legally clean.

Rae - the pixel artist

Comfortable in Aseprite. Wants generation for the boring 90% but full manual control for the last 10%, without leaving the tool.

Alex - the automator

Tech-lead/agent-builder. Wants to script generation, export, and engine sync in CI - the coding parts, not the art direction.

End-to-end scenarios

1 · Jam hero in five minutes (Jamie)

Jamie opens the character generator, picks a body, stacks clothes, recolors via tokens, previews the walk, exports a sheet with credits, and drops it into the engine - all before coffee cools.

2 · A whole coherent world (Sam)

Sam generates a cast of characters, paints a grass-and-dirt map with the environment generator, drops in houses and props, applies one theme so everything matches, exports the lot with lighting maps, and ships attribution automatically.

3 · Pirates and mounts (Sam)

Sam enables the Pirates pack and a horse mount pack, builds a mounted pirate captain (rider+mount compositing), and gets a correctly layered, animated unit without hand-aligning anything.

4 · Off-catalog custom art (Rae)

Rae generates a base character, opens a frame in the in-app editor, hand-paints a unique helmet with token-bound swatches, saves it into a pack, and reuses it - staying on-palette the whole time.

5 · Dynamically-lit night scene (Sam)

Sam builds a scene, relies on auto-generated normal/specular/emissive maps so torches and glowing runes light sprites correctly, checks it under a movable preview light, and exports to Unity's 2D lighting slots.

6 · Retheme for a new biome (Sam)

Sam duplicates a summer forest, rebinds the foliage and ground tokens to an autumn/dusk theme, and the entire project - characters, terrain, structures - recolors coherently in one move.

7 · Bulk NPC variants via CLI (Alex)

Alex writes a config listing 40 townsfolk recipes, runs the CLI in CI to batch-generate, export atlases, and refresh credits headlessly - no GUI, no manual repetition.

8 · Live iteration into Godot (Rae + Alex)

With the live link pointed at a Godot project, every edit - a recolor, a new frame, a tweaked normal map - appears in the running game instantly, turning iteration into a feedback loop.

Consolidated "user can…" catalog

Everything the proposed product lets a user do, grouped by area. This is the union of the per-proposal lists, deduplicated into one reference.

Characters

  • User can pick a body type and stack layers (body, head, hair, headwear, arms, torso, legs, feet, tools, weapons).
  • User can enable themed and community packs (e.g. pirates) as one-click add-ons.
  • User can build non-humanoid subjects: mounts, animals, and creatures.
  • User can compose rider + mount into one correctly layered, animated unit.
  • User can see per-animation coverage and which pack fills each slot.
  • User can register a custom pack of their own art that behaves like a built-in.

Environment & terrain

  • User can generate a seamless autotile set for any material (grass, dirt, sand, stone, wood, water) from one choice.
  • User can generate transition tiles between any two materials.
  • User can assemble a "biome" of materials that blend together.
  • User can control detail/scatter density and variation.
  • User can author animated terrain (water ripples, foam, waterfalls).
  • User can preview terrain by painting materials on a test map with live autotiling.

Structures & props

  • User can assemble buildings from parts (shell, roof, doors, windows, trim) that stay perspective-correct.
  • User can place and recolor props (furniture, clutter, nature) from packs.
  • User can set a footprint, anchor, and z-order so objects drop into a scene correctly.
  • User can give a prop an optional animation (flicker, sway).

Coherence (tokens)

  • User can recolor any asset through shared palette tokens, always staying on-ramp.
  • User can retheme an entire project by rebinding tokens (summer→autumn, day→dusk).
  • User can define and reuse custom themes across projects.
  • User can override specific tokens on one asset without breaking the shared theme.
  • User can inspect which tokens an asset uses and where a token is used project-wide.

Manual edit & create

  • User can open any generated asset and edit its pixels or frames in-app.
  • User can create a brand-new asset from scratch on the project grid.
  • User can draw with token-bound swatches and stay on-palette automatically.
  • User can author or extend animations with onion-skinning.
  • User can save hand-made art into a pack for reuse in the generators.
  • User never has to leave the tool for routine pixel work.

Material & lighting maps

  • User can generate a coordinated map set (normal, specular, emissive, +optional) with any asset.
  • User can preview an asset under a movable light to judge the maps.
  • User can toggle which maps an asset produces (lean for unlit games).
  • User can hand-edit any map channel, aligned to the diffuse.
  • User can rely on maps staying in sync across frames, atlases, recolors, and scenes.

Compose, export & automate

  • User can place characters, terrain, and structures together in one scene with correct depth sorting.
  • User can export packed files (atlas + metadata) for the target engine.
  • User can live-link a project so edits auto-update in the running engine (e.g. Godot).
  • User can drive generation, export, and sync headlessly via the CLI/API for agents and CI.
  • User can get correct, merged credits across every asset and pack in the project.
  • User can save, reopen, and share a project (and reproduce any build).

Task flows

Step-by-step journeys that cross multiple proposals. Each is a concrete sequence a user follows.

Flow A · Assemble & ship a character

  1. Choose subject & packs

    Pick body type; enable any themed packs (e.g. pirates).

  2. Compose

    Stack layers across slots; auto-match colors.

  3. Theme

    Recolor via tokens to fit the project palette.

  4. Preview

    Scrub animations; check coverage and pivots.

  5. Maps

    Generate normal/specular/emissive; check under a light.

  6. Export

    Send atlas + maps + credits to the engine (files or live link).

Flow B · Build a coherent, lit scene

  1. Paint terrain

    Choose a biome; paint materials with live autotiling and transitions.

  2. Place structures & props

    Drop buildings and objects; they sort by anchor/z-order.

  3. Add characters

    Place the cast; preview animations in situ.

  4. Unify the look

    Apply one theme; tokens recolor everything on-palette.

  5. Light it

    Rely on generated maps; verify with a movable preview light.

  6. Export together

    Emit the scene's assets + maps + merged credits for the engine.

Flow C · Customize off-catalog (generate → edit → reuse)

  1. Generate a base

    Assemble the closest character/prop from packs.

  2. Open the editor

    Edit a frame/layer in-app with token swatches.

  3. Create custom art

    Hand-paint the new piece; palette-locked, style-assisted.

  4. Author maps

    Tweak or paint the normal/emissive for the new art.

  5. Save to a pack

    Register it so the generator can reuse it later.

  6. Sync

    Live-link updates the running game immediately.

Flow D · Retheme a whole project

  1. Duplicate

    Copy the project (e.g. summer forest).

  2. Rebind tokens

    Point foliage/ground/water tokens at an autumn/dusk ramp set.

  3. Verify

    Everything - characters, terrain, structures - recolors coherently.

  4. Export the variant

    Ship the reskinned set alongside the original.

Flow E · Automate with the CLI (Alex)

  1. Author a recipe

    Describe variants/assets as a config/spec file.

  2. Batch-generate

    Run the CLI to produce every asset headlessly.

  3. Export & sync

    Emit atlases + maps; deploy into the project via live link.

  4. Refresh credits

    Regenerate and validate merged attribution.

  5. Run in CI

    Wire the whole flow into a pipeline or agent.

Why this matters

Every flow but the simplest touches several proposals - proof the value is in the connections. A user rarely wants "just a generator"; they want to go from idea to a coherent, lit, engine-ready, correctly attributed result. These scenarios and flows are the specification for that journey.
