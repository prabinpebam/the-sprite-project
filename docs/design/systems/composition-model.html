Design · Systems

How It Comes Together

The composition model that lets characters, terrain, and structures live in one project, share a look, and export together.

TL;DR

The three generators are not separate apps - they are producers that emit assets into a single project. A shared scene/composition layer places those assets on the same grid with correct depth sorting; a shared token style system keeps them coherent; a shared export pipeline (files, live link, CLI) ships them together. The glue is common data, not copied features.

One project, many producers

Everything a user makes belongs to a project. Each generator writes into it using the same underlying asset model, so a character, a grass tileset, and a house are siblings that already agree on grid, palette, and license format.

The shared asset model

Whatever the generator, an asset carries the same core fields so the rest of the system can treat them uniformly.

Grid & size
32px-aligned dimensions and footprint, so everything snaps to the same tilemap.
Anchor & z-order
Ground origin and depth rules for correct sorting between characters, props, and terrain.
Palette tokens
Recolorable regions bound to shared tokens (see token coherence).
Animation set
Frames/directions via one animation format used by all generators.
Material & lighting maps
Normal, specular, emissive, and related maps that stay in lockstep with the diffuse (see material & lighting maps).
License metadata
Per-source credits that merge across every asset in the project.

The scene: composing across generators

A scene is an optional composition surface where the three asset kinds meet: paint terrain, drop structures and props, place characters. Its value is verification and export context - seeing that a character reads against the ground and sorts correctly behind a roof, and exporting a coherent set rather than loose files.

  • Terrain forms the base layer via autotiling.
  • Structures and props sort by anchor/z-order against characters.
  • Characters preview their animations in situ.
  • The whole scene, or any selection, exports together.

Coherence and editing run through everything

Token style system

One palette-aligned token set styles every generator, so a project-wide recolor stays consistent across characters, terrain, and structures.

Manual editor

Any generated asset can be hand-edited in-app without leaving the token system, so custom work stays coherent and re-exportable.

One export pipeline

Because all assets share the model, a single pipeline serves every generator: packed files (atlas + metadata), an engine live link that writes native resources into a running project, and a CLI/API for agentic automation. Credits merge automatically across the whole project, and every asset's lighting maps export into the engine's native slots alongside the diffuse.

The unifying idea

The generators differ in what they make but agree on how assets are described. That shared description - grid, tokens, animation, license - is what lets three tools behave like one coherent product.
