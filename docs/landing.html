Build Now, Grow Toward the Vision

TL;DR

The documentation separates verified product scope, current architecture, and long-term direction. The MVP completes one character-to-Godot journey. ADR-001 fixes the sustainable delivery model: GitHub Pages, local browser storage, offline use, and no required server. The roadmap expands only through evidence-gated stages.

Review the locally evidenced UPD-002 · Local Content Pack Ecosystem. Actual Pages publication and independent clean-machine Windows observation remain release verification gates.

Two horizons, one product

MVP · Build now

  • One project-based LPC character generator
  • Pack-driven composition and animation preview
  • Token recoloring and reproducible recipes
  • Generic and Godot export with exact attribution

North Star · Grow toward

  • Characters, terrain, structures, and props
  • Project-wide visual coherence and manual editing
  • Scene composition and material maps
  • Engine links, CLI/API automation, and collaboration

How the MVP expands

The first product is deliberately narrow, but its versioned contracts become the center of each later expansion.

Working conventions

Now versus later
mvp/ is the active product boundary; north-star/ protects the complete destination.
Sequencing
planning/ contains delivery phases and evidence-based exit gates, beginning with implementation now.
Architecture
architecture/decisions/ preserves accepted rationale; architecture/platform/ defines the current storage, delivery, offline, and host contracts.
Product change control
governance/ defines the mandatory update pipeline; delivery/updates/ holds each versioned request, baseline, derivation, execution plan, and evidence.
Supporting knowledge
research/ is the evidence base; design/ contains detailed long-term proposals; execution/ records what is actually built.
Navigation
The sidebar is generated from folder paths, so adding a page or category needs no code - just a manifest entry.
Authoring
Pages are built with the docs-presentation (Slate) skill: body fragments, catalog components, visual-first.
