Long-term direction · North Star

A Coherent World-Asset Studio

One project where developers generate, adapt, compose, and ship an entire visually coherent 2D world without stitching together disconnected tools and asset sources.

TL;DR

The North Star is a family of asset producers for characters, environments, structures, and props, joined by one project model, token language, editor, provenance system, composition surface, and export pipeline. The MVP begins with character generation, but it establishes the shared contracts that let later producers join without becoming separate products.

North Star, not current scope

This page protects the destination while the delivery roadmap controls sequencing. Features described here are not commitments for the first release.

Current platform boundary

The North Star does not imply paid infrastructure. The accepted host strategy combines a complete zero-server web baseline with an optional portable Electron host. Both hosts are implemented through UPD-001. UPD-002 now proposes the local content-pack ecosystem; terrain and additional producers remain sequenced behind that evidence. A backend still requires a separate funded product update and ADR.

The complete product

The user promise

A developer can move from an idea to a coherent, animated, engine-ready world while retaining control over style, source attribution, and output. Generation handles repetitive construction; manual editing handles the exceptions; shared tokens keep every result visually related.

Product pillars

Extensible producers

Characters, terrain, structures, and props are produced through specialized workflows but enter the project through shared asset and pack contracts.

Coherence by construction

Semantic color and material tokens make project-wide visual consistency a property of the system rather than a manual cleanup task.

Generation with control

A token-aware editor lets artists change generated work, create missing assets, and feed those results back into reusable packs.

One project and pipeline

Assets share project, animation, provenance, composition, and export models so the product behaves as one studio rather than a bundle of generators.

Durable principles

One project
New producers join the same project; they do not become isolated applications.
Data before exceptions
Packs encode slots, layers, compatibility, animation coverage, palette mappings, and licenses.
Headless core
Composition and export remain usable without the graphical interface, enabling later automation.
Sustainable delivery
The complete required product runs from static GitHub Pages files with local user-owned data; infrastructure follows proven need.
Provenance never detaches
Authorship and license information travel with every source asset and every export.
Expand after evidence
Shared abstractions are extracted from working producers, not invented in anticipation of them.
