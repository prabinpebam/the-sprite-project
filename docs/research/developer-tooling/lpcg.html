lpcg - Liberated Pixel Cup Generator

A headless Rust library and CLI that merges LPC layers into a spritesheet from plain identifiers - generation as code, no browser.

TL;DR

By Bastien Sevajol (buxx), lpcg takes an ordered list of layer identifiers and produces a merged image. It is usable as a Rust crate (Builder + Input) or a CLI binary, supports random (*) and variant selection, and imports its spritesheets folder from the sanderfrenken ULPC repo via a Python script. It is small (4 stars), ~4 years old, and licensed AGPL-3.0.

How it works

Layers are addressed by ::-separated identifiers, merged in order. A * picks a random file; a (variant) suffix prefers a matching variant image if present.

# Merge body then head, in order
body::bodies::female::fur_tan head::heads::rabbit::fur_tan

# Random head, light variant across layers
body::bodies::*(light) head::heads::*(light) legs::pantaloons::*(light)

As a CLI:

lpcg-build ./spritesheets "body::bodies::*(light)" "head::heads::*(light)" "legs::pantaloons::*" out.png

As a Rust library:

use std::path::PathBuf;
use lpcg::{builder::Builder, input::Input};

let input = Input::from_str("body::bodies::female::fur_tan head::heads::rabbit::fur_tan").unwrap();
let image = Builder::new(PathBuf::from("./spritesheets")).build(input).unwrap();
image.save(&PathBuf::from("./out.png")).unwrap();

Where its art comes from

  1. Source of truth

    The upstream sanderfrenken Universal-LPC-Spritesheet-Character-Generator.

  2. Import

    import_from_ulpcscg.py copies only 832x1344 PNGs and universal folder content.

  3. Credits

    CREDITS.csv is regenerated automatically on import.

Relevance to this project

lpcg is the reference for programmatic, server-side or build-time sprite generation - the opposite end from the interactive web apps. Its identifier-based layer language and random/variant selection are a clean model for scripting bulk character generation.
