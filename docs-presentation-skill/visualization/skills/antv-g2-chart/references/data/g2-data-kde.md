## Core concepts

**KDE is a data transform, not a mark transform**

- Configure data transforms in `data.transform`
- Kernel density estimation is a nonparametric statistical method for estimating the probability density function of a random variable
- Internally, it uses the [pdfast](https://www.npmjs.com/package/pdfast) library

**Output**: the processed data adds two fields, defaulting to `y` and `size`, both of which are array types.

**Key point: `field` is the input, `as` is the output, and encode must use the output field names**

```
kde: field='value' -> output as=['y','size'] -> encode: { y: 'y', size: 'size' }
                      ^ no matter what field is named, encode always uses the values of as
```

## Minimal runnable example

```javascript
import { Chart } from '@antv/g2';

const chart = new Chart({ container: 'container', width: 700, height: 400 });

chart.options({
  type: 'density',
  data: {
    type: 'fetch',
    value: 'https://assets.antv.antgroup.com/g2/species.json',
    transform: [
      {
        type: 'kde',
        field: 'y',              // Field for KDE
        groupBy: ['x', 'species'], // Grouping fields
        size: 20,                // Number of generated data points
      },
    ],
  },
  encode: {
    x: 'x',
    y: 'y',
    color: 'species',
    size: 'size',
  },
  tooltip: false,
});

chart.render();
```

## Configuration options

| Property | Description                                                        | Type               | Default          | Required |
| -------- | ------------------------------------------------------------------ | ------------------ | ---------------- | -------- |
| field    | Data field used for the kernel density algorithm                   | `string`           | -                | yes      |
| groupBy  | Grouping fields used to group the data; multiple fields are allowed | `string[]`         | -                | yes      |
| as       | Fields used to store the data after KDE processing                 | `[string, string]` | `['y', 'size']`  | no       |
| min      | Minimum value of the processing range                              | `number`           | Data minimum     | no       |
| max      | Maximum value of the processing range                              | `number`           | Data maximum     | no       |
| size     | Number of rows generated by the algorithm; larger values make the density curve more detailed | `number` | `10` | no |
| width    | Determines how many neighboring points each element influences; larger values make the curve smoother | `number` | `2` | no |

## Violin plot

```javascript
chart.options({
  type: 'view',
  data: { type: 'fetch', value: 'https://assets.antv.antgroup.com/g2/species.json' },
  children: [
    {
      type: 'density',
      data: {
        transform: [
          { type: 'kde', field: 'y', groupBy: ['x', 'species'] },
        ],
      },
      encode: { x: 'x', y: 'y', series: 'species', color: 'species', size: 'size' },
      tooltip: false,
    },
    {
      type: 'boxplot',
      encode: { x: 'x', y: 'y', series: 'species', color: 'species', shape: 'violin' },
      style: { opacity: 0.5, strokeOpacity: 0.5, point: false },
    },
  ],
});
```

## Custom parameters

```javascript
chart.options({
  type: 'density',
  data: {
    transform: [
      {
        type: 'kde',
        field: 'y',
        groupBy: ['x'],
        size: 30,             // More detailed density curve
        width: 3,             // Smoother
        min: 0,               // Minimum value
        max: 8,               // Maximum value
        as: ['density_x', 'density_y'],  // Custom output field names
      },
    ],
  },
  encode: { x: 'x', y: 'density_x', size: 'density_y' },
});
```

## Common errors and fixes

### Error 1: Putting kde in a mark transform

```javascript
// Incorrect: kde is a data transform and cannot be placed in a mark transform
chart.options({
  type: 'density',
  data,
  transform: [{ type: 'kde', field: 'y' }],  // Incorrect location
});

// Correct: put kde in data.transform
chart.options({
  type: 'density',
  data: {
    type: 'fetch',
    value: dataUrl,
    transform: [{ type: 'kde', field: 'y', groupBy: ['x'] }],  // Correct
  },
});
```

### Error 2: Missing groupBy

```javascript
// Incorrect: groupBy is required
data: {
  transform: [{ type: 'kde', field: 'y' }],  // Missing groupBy
}

// Correct: specify groupBy
data: {
  transform: [{ type: 'kde', field: 'y', groupBy: ['category'] }],
}
```

### Error 3: Zero variance or single-point data causes KDE to degenerate, with silent blank output

The underlying KDE implementation, the pdfast library, requires `min < max`. When all values in a group are identical (variance = 0) or the group has only 1 data point, KDE produces NaN and that group is not rendered.

```javascript
// Problem: zero-variance data
const data = [
  { group: 'A', value: 10 },           // Only 1 point
  { group: 'B', value: 20 },
  { group: 'B', value: 20 },           // All identical, min=max=20
  { group: 'B', value: 20 },
];
// When KDE processes group B with groupBy: ['group'], min=max=20, so it produces NaN and does not render

// Solution 1: manually set min/max to ensure a nonzero range
transform: [{
  type: 'kde',
  field: 'value',
  groupBy: ['group'],
  min: 0,    // Manually specified to ensure min != max
  max: 100,
}]

// Solution 2: use a different chart type when data is insufficient
// KDE works best when each group has at least 5 to 10 distinct values to produce a meaningful density curve
// Single points or identical values -> use a scatter plot or boxplot instead
```

### Error 4: encode fields do not match as

```javascript
// Incorrect: default as is used, but encode uses other field names
data: {
  transform: [{ type: 'kde', field: 'y', groupBy: ['x'] }],  // Default as: ['y', 'size']
}
encode: { y: 'density', size: 'density_size' },  // Incorrect field names

// Correct: use the correct field names
encode: { y: 'y', size: 'size' },  // Uses the default output fields

// Or customize as
 {
  transform: [{ type: 'kde', field: 'y', groupBy: ['x'], as: ['density', 'density_size'] }],
},
encode: { y: 'density', size: 'density_size' },  // Matches custom fields
```

### Error 5: Insufficient data causes empty or invalid KDE results

KDE requires enough data points to generate a valid density estimate. If a group has too few data points, such as 1 or fewer, KDE cannot calculate a valid result and the group may not be displayed.

```javascript
// Problem: some groups have insufficient data
const data = [
  { species: 'setosa', x: 'SepalLength', y: 5.1 },
  { species: 'versicolor', x: 'SepalLength', y: 6.0 },
  { species: 'virginica', x: 'SepalLength' }, // Missing y value
];

// Solution: ensure each group has enough valid data
const validData = [
  { species: 'setosa', x: 'SepalLength', y: 5.1 },
  { species: 'setosa', x: 'SepalLength', y: 5.0 },
  { species: 'versicolor', x: 'SepalLength', y: 6.0 },
  { species: 'versicolor', x: 'SepalLength', y: 6.2 },
  { species: 'virginica', x: 'SepalLength', y: 6.5 },
  { species: 'virginica', x: 'SepalLength', y: 6.3 },
];
```

### Error 6: Incorrect chart type or encoding causes rendering failure

When using KDE in a combined view, make sure every child chart correctly sets its type and encoding, especially the `series` mapping.

```javascript
// Incorrect: required encode fields are missing or the type setup is improper
children: [
  {
    type: 'density',
    data: { transform: [{ type: 'kde', field: 'y', groupBy: ['x', 'species'] }] },
    encode: { x: 'x', y: 'y', color: 'species', size: 'size' },
  },
  {
    type: 'boxplot',
    encode: { x: 'x', y: 'y', color: 'species', shape: 'violin' }, // Missing series mapping
  }
]

// Correct: ensure all required fields are correctly mapped
children: [
  {
    type: 'density',
    data: { transform: [{ type: 'kde', field: 'y', groupBy: ['x', 'species'] }] },
    encode: { x: 'x', y: 'y', series: 'species', color: 'species', size: 'size' },
  },
  {
    type: 'boxplot',
    encode: { x: 'x', y: 'y', series: 'species', color: 'species', shape: 'violin' },
  }
]
```

### Error 7: Too few data points after KDE grouping renders a blank chart

KDE requires each group to have enough data points to generate a valid density estimate. If too few data points remain after grouping, such as fewer than 2 valid values per group, KDE cannot calculate a valid result and may render the entire chart blank.

```javascript
// Problem: each group has too few data points after grouping
const insufficientData = [
  { species: 'setosa', x: 'SepalLength', y: 5.1 },
  { species: 'versicolor', x: 'SepalLength', y: 6.0 },
  { species: 'virginica', x: 'SepalLength', y: 6.5 },
];

// Solution: ensure each group has enough valid data points
const sufficientData = [
  { species: 'setosa', x: 'SepalLength', y: 5.1 },
  { species: 'setosa', x: 'SepalLength', y: 5.0 },
  { species: 'setosa', x: 'SepalLength', y: 5.2 },
  { species: 'versicolor', x: 'SepalLength', y: 6.0 },
  { species: 'versicolor', x: 'SepalLength', y: 6.2 },
  { species: 'versicolor', x: 'SepalLength', y: 5.9 },
  { species: 'virginica', x: 'SepalLength', y: 6.5 },
  { species: 'virginica', x: 'SepalLength', y: 6.3 },
  { species: 'virginica', x: 'SepalLength', y: 6.7 },
];
```

### Error 8: encode is not set correctly when KDE is used with the density chart type

When using KDE as a data transform, make sure `encode` correctly references the fields generated by KDE, defaulting to `y` and `size`. Otherwise, the chart may not render correctly.

```javascript
// Incorrect: fields generated by KDE are not referenced correctly
chart.options({
  type: 'density',
  data: {
    transform: [{ type: 'kde', field: 'y', groupBy: ['x'] }]
  },
  encode: { x: 'x', y: 'originalY', size: 'originalSize' } // Incorrect field names
});

// Correct: reference the fields generated by KDE
chart.options({
  type: 'density',
  data: {
    transform: [{ type: 'kde', field: 'y', groupBy: ['x'] }]
  },
  encode: { x: 'x', y: 'y', size: 'size' } // Correct field names
});
```

### Error 9: Poor KDE grouping field selection causes abnormal rendering

When using KDE, choosing `groupBy` fields is very important. Selecting the wrong fields or failing to fully account for the data structure can make the generated density curve fail to match expectations or become completely invisible.

```javascript
// Incorrect: groupBy fields are poorly chosen
chart.options({
  type: 'density',
  data: {
    transform: [
      {
        type: 'kde',
        field: 'y',
        groupBy: ['species'], // Groups only by species and ignores different categories in the x field
      },
    ],
  },
  encode: {
    x: 'x',
    y: 'y',
    color: 'species',
    size: 'size',
  },
});

// Correct: choose groupBy fields appropriately
chart.options({
  type: 'density',
  data: {
    transform: [
      {
        type: 'kde',
        field: 'y',
        groupBy: ['x', 'species'], // Group by both x and species
      },
    ],
  },
  encode: {
    x: 'x',
    y: 'y',
    color: 'species',
    size: 'size',
  },
});
```

### Error 10: KDE output fields are overwritten by later operations

When fields generated by KDE, such as `y` and `size`, are modified or overwritten in later data processing steps, chart rendering can become abnormal.

```javascript
// Incorrect: KDE output fields are overwritten by a later transform
chart.options({
  type: 'density',
  data: {
    transform: [
      { type: 'kde', field: 'y', groupBy: ['x', 'species'] },
      { type: 'map', callback: (d) => ({ ...d, y: d.y.map(v => v * 2) }) } // Modifies the y field
    ],
  },
  encode: {
    x: 'x',
    y: 'y',
    color: 'species',
    size: 'size',
  },
});

// Correct: use custom field names to avoid conflicts
chart.options({
  type: 'density',
  data: {
    transform: [
      { 
        type: 'kde', 
        field: 'y', 
        groupBy: ['x', 'species'],
        as: ['kdeY', 'kdeSize'] // Use custom field names
      },
      { type: 'map', callback: (d) => ({ ...d, y: d.y.map(v => v * 2) }) } // Does not affect KDE results
    ],
  },
  encode: {
    x: 'x',
    y: 'kdeY', // Use the custom field generated by KDE
    color: 'species',
    size: 'kdeSize',
  },
});
```
