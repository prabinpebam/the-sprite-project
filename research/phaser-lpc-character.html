Phaser 3 Universal LPC Character

A TypeScript integration that turns LPC spritesheets into ready-to-use game objects for the Phaser 3 engine.

TL;DR

By ivopc, this project wraps Universal-LPC art for Phaser 3, exposing characters as Phaser.GameObjects.Sprite, Phaser.Physics.Arcade.Sprite, and Phaser.Physics.Matter.Sprite, with animators, input behaviors, and typed interfaces for every spritesheet variation. It is TypeScript-first, Vite-built, GPL-3.0, and openly under development (8 stars).

What it offers

Multiple game-object flavors

Plain sprite, Arcade-physics sprite, and Matter-physics sprite - so the same character drops into whichever physics system a game uses.

Animators & input

Built-in animation handling and input behaviors wired to the LPC animation set.

Typed variations

The full range of spritesheet character variations expressed as TypeScript interfaces and methods - discoverable and type-safe.

Component architecture

Prefab components, traits interfaces, and procedural dynamic traits for assembling characters in code.

Getting started

# install
npm i
# start dev server
npm run dev
# build
npm run build
# generate the spritesheet JSON map
npm run generate-spritesheet-map

Relevance to this project

This shows the "last mile" - taking LPC output and making it engine-native with typed APIs and physics bodies. It is a model for how a sprite tool's output could ship with first-class engine bindings rather than bare PNGs.
